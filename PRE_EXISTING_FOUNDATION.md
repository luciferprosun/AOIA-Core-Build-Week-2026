## Conservatively supported foundation

The baseline repository already described AOIA-Core as a local-first research prototype and safety core, not a production autonomous-agent runtime. Committed baseline documentation and tests supported narrow claims around pre-execution inspection, proposal review, deterministic command-safety decisions in tested paths, human-approval separation, audit metadata, provenance/evidence boundaries, and controlled model-routing work.

The baseline also contained extensive historical and transitional material. Its own documentation warned that legacy execution, browser, provider, filesystem, Git, and canonical-promotion surfaces were not globally closed. Documentation or historical artifacts were not implementation authority. Any inert Memory Hat metadata present before the period was non-executing and is not activated here.

## What this file does not claim

It does not reconstruct every pre-period capability, validate every historical report, or state that the baseline was production-ready. It exists to prevent later Build Week work from being misrepresented as a one-week creation of the whole project.
