## Days 1–14 — stabilize and prove one bounded HAT bridge

- Complete the activities in the [14-day roadmap](14_DAY_ROADMAP.md).
- Preserve the current authority model while validating one German employment-law evidence adapter.
- Require exact provenance, bounded excerpts, explicit unavailable states, and independent operation from model observers.

Decision gate: continue only if the pinned implementation, focused tests, corpus provenance review, and manual demo agree.

## Days 15–30 — evidence lifecycle and evaluator depth

- Define versioned HAT descriptors, corpus identity, effective-date metadata, and deterministic freshness checks.
- Add conflict handling for multiple official sources without silently selecting authority.
- Build offline evaluation sets for temporal accuracy, insufficient evidence, ambiguous jurisdiction, and adversarial retrieved text.
- Document update and rollback procedures without granting the runtime corpus-write capability.

Decision gate: publish evaluation evidence only after human review; do not promote a score to approval.

## Days 31–45 — reliability and operator observability

- Exercise malformed provider responses, model substitution, observer failure, retrieval failure, stale evidence, cancellation, and finalizer failure.
- Make call order, model identity, evidence identity, and final-delivery state inspectable without exposing secrets or raw private conversations.
- Compare observer-only, HAT-only, and combined workflows on the same bounded scenarios.
- Review accessibility, keyboard operation, narrow-screen behavior, and recovery from interrupted sessions.

Decision gate: retain fail-closed behavior and document every unresolved limitation that affects a demo or reviewer claim.

## Days 46–60 — controlled reviewer release preparation

- Re-run clean-clone and reproducibility checks from a pinned release candidate.
- Reconcile source commits, generated summaries, copied evidence hashes, licenses, privacy boundaries, and public links.
- Prepare reviewer navigation and a short reproducible demonstration that distinguishes implemented features from planned extensions.
- Obtain separate human decisions for repository visibility, release packaging, deployment, and any production change.

Decision gate: release only the exact reviewed commit. Otherwise publish a blocked/not-ready status without simulated success.

## Continuing constraints

- Provider, observer, HAT, retrieval, roadmap, test, and manifest output remains non-authoritative.
- Human approval remains explicit, scoped, and separate from evidence generation.
- No legal decision, autonomous execution, silent network retrieval, automatic fallback, or unrestricted corpus import is planned.
- Production AOIA-Core, desktop-demo, and reviewer repositories remain separately versioned and independently reviewable.
